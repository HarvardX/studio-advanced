When might you need CSS?

There will be times times when a digital platform cannot do what you want it to do, and so you reach to the HTML editor to expand the functionality repertoire available to you. But, if you find yourself copy-and-pasting chunks of code between pages multiple times, because you’ve made something really useful, it might be worth considering developing a style sheet. This is particularly helpful when you’ve developed a ‘critical mass’ of HTML code chunks that you regularly reach for in your courses. 

Simply put, CSS is a method for standardising a style or approach of a given webpage, or a course in our case. CSS is the style and HTML the content. CSS uses HTML in the file for creating the style but then you use CSS short-hand references to streamline your in-page HTML code.

CSS comes into its own when creating more complex items in HTML, especially those with multiple lines of code. EdX doesn’t really like lots of these complex code items on the page directly - the viewer interface can get a bit glitchy - so try not to overdo it!

Whilst the upfront work can be a little higher to get everything looking as you wish, it will save a lot of time in the long run and means you can tweak your design without having to manually change the code on each page.

Pros and Cons

Benefits
  • Saves time: once the .css file is referenced on page, you only need to use short references in your page editor to elicit the desired style
  • Saves pain: any change to the core style sheet only needs to be made in the .css file, then uploaded again to edX, rather than manual changes to every instance of that style on your pages - the amended style 'cascades' across all pages
  • Provides wide design scope: when HTML gets more complex, CSS provides improved operation and scope
  • Improves load time of pages if they have lots of formatting enabled
  • Improved accessibility through consistency of style and ability to test and make adjustments to all course pages
  • Envy of all your friends as your course (and your code) will look awesome
Costs
  • Upfront time required to work out the design features required
  • New habit forming: it will feel a bit slow to begin with as you need to add new elements to each page as you make them
