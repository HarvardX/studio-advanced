Here's how it works: we create a variable in a Python script. Into that variable we place a random number. Then we reference that number as the answer.

<problem>

<script type="loncapa/python">
secretAnswer = random.randint(1, 11)
</script>

<numericalresponse answer="$secretAnswer">
  <label>What number am I thinking of? Is it $secretAnswer? You'll never guess.</label>
<formulaequationinput/>
</numericalresponse>

  <solution>
    <div class="detailed-solution">
      <p>How did you know???</p>
    </div>
  </solution>
  
</problem>

The variable is secretAnswer, which we refer to as $secretAnswer in the text of the problem and in the answer field.

This was just a toy example, but you can do some pretty great stuff with this:

  • Create random numbers in whatever range is appropriate for your problem.
  • Randomize some numbers and not others.
  • Do calculations in your Python to get an answer based on multiple random numbers.
  • Assign a random block of text by filling your variable with a string instead of a number.
  • Give out different web links to different students.
  • Show a random image, by inserting your variable into a URL. For example, <img src="/static/img$secretAnswer.png"> would pull a randomly numbered image from img1.png to img10.png.

It takes a little longer to write these problems, and you have to learn a tiny bit of coding, but in exchange you get much more robust assignments. Just remember to tweak the settings so that the problem randomzes on reset, and the reset button is always visible.

Warning: Not every problem type can be randomized.

There are some problem types where you can't easily randomize the answers - edX doesn't substitute $variables in every tag. Be sure to test your problems as you build them, and test again before they go live.
