Javascript

One of edX's missing features, from a data scientist's point of view, is an event stream that can be used to trigger specific responses. Luckily, we have a workaround! Dan Davis contributed this section on realtime data tracking, which he developed for research related to his doctoral thesis (along with a number of other papers).

Warning Security Hazard

The methods described in this section obtain student data for you. What you do with that data is up to you. We recommend doing something smarter than "store it unencrypted on a web server". Take appropriate measures to safeguard your learners' data.

The goal: Gather specific data about edX learners in real-time as they interact with a course.
Prerequisites: Javascript and some database experience.
Project length: At least a day to build and test.
Ingredients:

  • Access to Studio
  • A secure location in which to store learner data (not needed for initial testing).
