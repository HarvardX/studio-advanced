Flowchart of success

The hardest part of all of using CSS is deciding what you want to design and how you wish it to display. Once you've worked that out, the rest is straightforward. Consistency of code across your course pages is pretty rewarding in the long run (even if it feels like a bit of a slog initially, trust us!).

So, you've just made a great .css file. What next?

First, you need to upload the file into your edX course. This is just like any other file upload in Studio:

Content > Files & Uploads > Upload file

Then you need to reference the use of the CSS on each page (unit). There isn't an "apply to all pages" selection on the platform, nor would that always be a good idea (e.g. sometimes you want to use different CSS and would then need a complex 'exceptions' method) so you need to add this on manualsp;

In order to summon the style sheet, use the following code line in a 'Raw HTML' component, ideally at the top or bottom of your page:

<link rel="stylesheet" type="text/css" href="/static/example.css">

Remember to change the edx /static/ studio file reference, in the code above, to your own .css file. Also, remember that you must use a Raw HTML component, not just a text component. Follow this link to the the edX documentation if you don't remember how to switch between raw and visual editors.

Then go about your normal page creation. If you have created a new feature such as 'purplebox' or 'hx-underline', these will need to be added into the HTML of the text component on page, for example:

<h4 class="hx-underline">Something I now know</h4>
<div class="hx-purplebox">
<p>It's pretty cool that this stuff is a lot easier than I thought</p>
</div>

In most cases, your style will be visible in the studio editor interface once you save your text/code. However, what you see in Studio doesn't always look exactly the same as you would expect on the 'live' view, so take a look at the 'preview' to get a better sense of the finished item.

Once happy, publish as normal.

Not happy or not displaying correctly? Go back to your .css file, make changes as required, then upload again to the file area. No need to change any references on your pages, the style will update automatically - though sometimes you'll need to do a 'hard refresh' on your page to get the style sheet to reload.
