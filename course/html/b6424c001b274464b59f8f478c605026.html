So if you like the intro, you're going to love this.

MIT has a grading library called MITxGraders. Here's what you can do with it:

  • Change credit based on attempt number
  • Grade lists of things smartly
  • Handle input groupings with flexible ordering
  • Use multiple answers with feedback and partial credit (even with lists!)
  • Require minimum word counts in text responses
  • Validate text input

If you use math questions in your course, the library really shines here:

  • Grade vectors and matrices, and handle matrix algebra
  • Give automatic partial credit on formula problems
  • Grade intervals with open and closed ends
  • Handle a much larger list of math functions
  • Allow functions like \(f(x)\) and \(g(z)\) in learner answers
  • Control allowed student input (for example, you can disallow '*' so multiplication doesn't work, or blacklist certain functions)
  • Better handle complex variables
  • Grade for things other than equality (like equality mod 2*pi, or in a given range, or in the span of a set of basis vectors, or proportional to a vector, or...)

Of particular note is that all of the library functionality works together seamlessly. It also handles all the edge cases, assigns partial credit carefully, and strives to present meaningful error messages to students whenever something goes wrong.

They even have a complete example course where you can test things out.
