Catalog Settings

These settings affect the way your course appears (or doesn't) for registration. A lot of what's on this list is defunct or unnecessary when it comes to edX.org, but may still be in use if your org uses an Open edX distribution.

Default: "images_course_image.jpg"

Other settings: any png or jpg file, in quotes.

Edit the name of the course image file. You must upload this file on the Files & Uploads page. You can also set the course image on the Schedule & Details page.

Not much to say here, except that this and edX.org's Publisher don't always play nice with each other. You should probably ensure that you have the same image in both places.

The required image size for edx.org is 2120 pixels wide by 1192 pixels tall. Older sources may say 378 x 225 px, which is outdated.

Default: null

Other settings: any text

Enter the text that you want to use as the advertised starting time frame for the course, such as "Winter 2018". If you enter null for this value, the start date that you have set for this course is used.

In addition to listing vague dates to give your development team a little wiggle room, you can also use this to do a "soft launch", wherein your course is actually open for a day or two before the advertised start date. Note that learners will still receive automated e-mails from edX, so this will only work well if you don't use Section Highlights - and those are probably more useful than a soft launch, all things considered.

Default: null

Other settings: Any date in MM/DD/YYYY format.

Enter the date to announce your course.

This is not used on edX.org, having been replaced by settings in Publisher.

Default: "images_course_image.jpg"

Other settings: other things

Edit the name of the banner image file. You can set the banner image on the Schedule & Details page.

Older edX distributions separated the Course About Page Image (which appeared at the top of the About page) from the Course Banner Image (which appeared in the catalog). If your Open edX distribution uses this setting, the proper dimensions are 608 x 211 pixels.

Default: { "instructors": [] }

Other settings: JSON-formatted data

Enter the details for Course Instructor

We'll be honest with you, we don't even know the right format to use here. This is years out of date, especially on edX.org, where Publisher handles all of it.

Default: null

Other settings: true or false

Enter true or false. If true, the course appears in the list of new courses on edx.org, and a New! badge temporarily appears next to the course image.

This only affects courses that are offered on catalog systems other than edX.org.

Default: []

Other settings: A list of strings, like ["thing 1", "thing 2", "thing 3"]

Specify what student can learn from the course.

If you're on edX.org, this is another out-of-date item, handled by Publisher.

Default: null

Other settings: any integer

Enter the maximum number of students that can enroll in the course. To allow an unlimited number of students, enter null.

Useful for small, private courses where you don't want to accidentally have a thousand enrollees when someone posts the link on Reddit. Do people still use Reddit? Slashdot? Lycos? People still think AltaVista is great, right?

Default: "images_course_image.jpg"

Other settings: Any jpg or png

Edit the name of the video thumbnail image file. You can set the video thumbnail image on the Schedule & Details page.

You've got a course video? Great, it has a thumbnail. You probably want to provide a custom one in case your video accidentally shows a frame where someone has their mouth open while chewing, or someone caught in mid-crossfade, or a random jet-black frame. Upload your image, list it here, done.

Default: "both"

Other settings: "about" or "none"

Defines the access permissions for showing the course in the course catalog. This can be set to one of three values: 'both' (show in catalog and allow access to about page), 'about' (only allow access to about page), 'none' (do not show in catalog and do not allow access to an about page).

If you want to hide your courses, MOOCs may not be for you. Then again, I suppose we don't all make MOOCs. Anyway, hiding your course by having the release date in the future is probably the better approach.

Default: false

Other settings: true

Whether to restrict enrollment to invitation by the course staff.

Setting your start date in the future will do this as well. We don't recommend a multilayered defense against people signing up for your course, because that's really going to end up as multilayered "trying to figure out what setting we forgot to change and why can't people sign up oh no why did we do this to ourselves."

Default: false

Other settings: true

Enter true or false. If true, the course will be available to mobile devices.

I'll be honest with you, I'm pretty sure this setting is broken. The default setting on edX.org is "false", which would mean that their mobile app doesn't show any courses, which clearly is not what happens. Check with your Open edX administrator to see whether you need to pay attention to this setting.
