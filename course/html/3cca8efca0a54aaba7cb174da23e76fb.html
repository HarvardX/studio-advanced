Partial Credit

In addition to randomizing MCQs, setting numerical tolerances, and alternate answers, you can also give certain questions partial credit! The official edX docs give full detail on this, which you can award for checkbox problems, multiple-choice problems, numerical problems, and custom python-based problems (including JSInput). You can write your problem in markdown, then use the Advanced Editor to add partial_credit="points" to the problem-type tag. Each one has its own options:

  • Multiple Choice: You can set partial_credit="points", which lets you decide how many points each answer is worth.
  • Checkbox: You can set partial_credit="edc", short for "every decision counts", in which case you've essentially turned the checkboxes into a set of true/false questions. Or, you can set partial_credit="halves", which cuts the score in half for each wrong answer, or eventually drops it to zero if they get enough wrong.
  • Numerical: You can set partial_credit="close" to give partial credits for answers that are close to the correct one. Or, you can set partial_credit="list", where you give it a list of alternate answers that get some points (for example, off by a factor of 2 or 1/2). You can even do partial_credit="close,list" for both!
  • Custom Python: When you return your grading object, setting 'ok': 'Partial' defaults to 50% credit, or you can set the 'grade_decimal' value directly.

Again, check the official edX docs for all the precise syntax.
