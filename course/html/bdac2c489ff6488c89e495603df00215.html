How to make pretty HTML

  1. Keep it clean - use Raw HTML

    When creating new content, rather than reaching first for the Rich Text, always go for Raw HTML. This will certainly seem scary at the start and will take a little time to get used to but it's worth the investment.

    Paste text directly into a <p></p> and it will paste as plain text, no formatting. This will save a lot of time stripping out formatting. Likewise, if you have a block of formatted text that you wish to clean, copy and paste straight into a Raw HTML component and just add the <p></p>. Simple.

    It is also the only way you can include CSS references. CSS and Raw HTML components go hand-in-hand.

  2. Keep it lowercase

    For readability reasons, your HTML tags should be written in lowercase. It is possible to create content using <UPPERCASE TAGS> and will have minimal impact on load speed, but it will look odd, some people won't be able to read it and is not standard practice. Keep it simple, keep it lowercase.

  3. Use CSS

    HTML is for content, CSS for style. Try not to mix up the two.

    In-line style formatting within your HTML will take your page longer to load and make your code less accessible. Remember, you never know if someone will 'inspect page' and judge you for it! ;)

  4. Limit CSS

    This is unlikely to apply too much here as there are quite a few limits on what you can do to edX course CSS, however do bear in mind that any page that references CSS in the HTML will require cross-reference to your CSS file. If the file is massive with loads of specific style references for one use in a specific context, it will take longer for the broswer to render the correct style reference. Again, simplify and minimise the volume of CSS to improve general functionality.

  5. Consider other tips below
    • Use separate lines
    • Remove redundant references
    • Use clean, short names for links
    • Create <fig> containers rather than plain <img> images
