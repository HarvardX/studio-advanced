Writing New Problem Types with JSInput


Hard

Day

Javascript

You might have learned how to select the problem types that are best for your course already. But what if you really need something that's not already covered? Maybe you have a great idea for an authentic assessment or activity and you'd like to have it graded. What then?

If you have access to your edX server, you can write an XBlock, but a) not all of us have that access, and b) sometimes an XBlock is overkill. Instead, you might want a Custom Javascript Display and Input Problem (also known as a JSInput Problem). The official documentation is good, but we're going to walk through a few implications and key points that will let you create one with more confidence and with a better idea of this problem type's capabilities.

The goal: Write a custom javascript problem.
Prerequisites: HTML, CSS, and Javascript.
Project length: At least a day to build and test.
Ingredients:

  • If you have existing resources that you want to use - graphics, Javascript libraries, etc. - you should get those together first.
  • We're going to refer to sample code from Stanford's JSInput Repository at GitHub.
