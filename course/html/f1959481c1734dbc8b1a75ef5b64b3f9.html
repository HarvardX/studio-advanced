Assignment Settings (Course)

This section covers assignment-related settings like gradebooks and special exam types.

Default: true

Other settings: false

If this value is true, learners can choose to take proctored exams without proctoring. If this value is false, all learners must take the exam with proctoring. This setting only applies if proctored exams are enabled for the course.

Exam proctoring is typically only used for high-stakes courses where verification is crucial. However, they also require a lot from the learners: a webcam, a secure environment, and the ability to install software. Learners in computer cafes, using library computers, or taking the course on a mobile device will be unable to take proctored exams. Since learners who opt out of proctoring must already acknowledge that this option makes them ineligible for course credit, you should probably leave this option as "true". Otherwise, you're just locking less fortunate learners out of parts of your course.

Default: false

Other settings: true

If true, the course will not be graded.

This setting is rarely used. If you know for certain that you do not want any part of the course graded and do not intend to issue any certificates, you might consider using this setting.

Default: null

Other settings: Any date in MM/DD/YYYY format.

Enter the date by which problems are due.

This sets a single due date for your entire course. Instructor-paced courses generally set due dates for each section individually, and will not need to use this setting. Self-paced courses have the end of the course as their due date by default, and learners will expect that they can work up until the last minute. You should not use this setting unless you have clear and frequent messaging to learners reminding them of the due date.

See also "Due Date Display Format"

Default: false

Other settings: true

If this value is true, you can hide a subsection until learners earn a minimum score in another, prerequisite subsection.

You cannot set a maximum score, and you cannot set any alternative content that learners will see if they fail to obtain the prerequisite score. Each subsection can have one and only one prerequisite, but you can chain them together if you want.

Because there is no down side to activating this setting, we recommend that you set this to "true". However, we also recommend that you do not use this setting to prevent learners from moving forward in your course. Blocking learners from progressing does nothing but remove them from your course.

Instead, be playful and inventive with your uses. Hide fun rewards instead of course content. Build games instead of walls.

Default: false

Other settings: true

If this value is true, timed exams are enabled in your course. Regardless of this setting, timed exams are enabled if Enable Proctored Exams is set to true.

We do not recommend using timed exams in MOOCs or other distance education settings, particularly for reasons of access. Some learners need extra time, language issues slow down many learners, and technological issues can intervene as well. If your client is insistent on using timed exams, you should at least double the amount of time that would normally be provided on-campus.

If you are running an in-person course, timed exams are a more normal part of the educational experience. In such cases, you can set this to "true". You will still need to give out extensions from time to time, so be sure that whomever is proctoring the exam has staff access to the course.
