Wrapping it up

You can use this approach to package the data however you like, to send to a server or database via an Ajax call. Back-end definitely isn’t my specialty, so I’ll leave that to the experts. But this at least equips you with the front-end hacking skills to start exploring more and thinking about how best to leverage this tracking approach into something that you can use to store and retrieve stored learner data in real-time.

Warning Workload Hazard

The approach described here must be applied to every course page (vertical, in edX terms) from which you want to gather data. This might mean every page in the entire course. This is inevitably going to be a time consuming process until there is a way to automate this, and/or until edX affords the option of inserting course-wide code in a course.
