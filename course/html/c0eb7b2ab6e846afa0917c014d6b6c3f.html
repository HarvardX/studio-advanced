Miscellaneous Settings

Default: "" (a blank string)

Other settings: a URL

URL for CCX Connector application for managing creation of CCXs. (optional). Ignored unless 'Enable CCX' is set to 'true'.

If you're using CCX, you probably already know what to do with this. If not, you can safely ignore it.

Default: null

Other settings: an integer

Enter the number of days before the start date that beta users can access the course.

Why in the world does this default to null? Set this to a reasonable number. 10000 days or something like it. Your beta testers should be able to get into the course before it opens.

Note that, for courses with feature-based enrollment, beta testers see all content regardless of gating settings, and do not lose access after the course duration has run out.

(Note also that Colin did once break a course by setting this number to... well, probably over a quadrillion, he just typed 1 and then held down the 0 and probably ran off the end of a Python integer or something. So don't go too crazy here.)

Default: false

Other settings: true but honestly why.

Enter true or false. If true, students cannot view the progress graph.

Perhaps someday in the future there will be good ways to create alternative grading displays in edX, and thus a good reason to shut off the Progress page. Until then, leave this as false.

Default: false

Other settings: true

Allow course instructors to assign CCX Coach roles, and allow coaches to manage Custom Courses on edX. When false, Custom Courses cannot be created, but existing Custom Courses will be preserved.

CCX is beyond the scope of this course. If you're using it already, this will be true; if not, just keep it as false.

Default: false

Other settings: true

If true, you can use the LaTeX templates for HTML components and advanced Problem components.

Note that this isn't using LaTeX to create equations on an HTML page. That's already enabled. This is using LaTeX to build things instead of XML. There's not much good reason to do this unless you love LaTeX and refuse to learn XML, so we recommend leaving this as false. If you have a love for LaTeX and unsupported features, you can check out Problem Written in LaTeX in the edX docs and Ike Chuang's latex2edx converter.

Default: null

Other settings: a URL

Enter the external login method students can use for the course.

Talk to your Open edX administrator to find out whether you're using this feature. If not, don't touch it.

Default: null

Other settings: a URL

If dashboard social sharing and custom course URLs are enabled, you can provide a URL (such as the URL to a course About page) that social media sites can link to. URLs must be fully qualified. For example: http://www.edx.org/course/Introduction-to-MOOCs-ITM001

If you encourage your learners to share your courses on social media, this will do a little to help out. The down side: you can't just drop in a URL yourself; you need your site administrator to add something to a file in the source code. Check out Sharing your course on social media at the edX help center to see what it looks like, and Enabling Social Sharing of Courses and Certificates for technical details.

Default: "" (a blank string)

Other settings: A URL

Enter the path to use for files on the Files & Uploads page. This value overrides the Studio default, c4x://.

If you'd rather upload files to an Amazon box or whatever, you can insert a URL above to point at it. For course builders, you'll still be using /static/filename.wtf to reference the images in Files & Uploads. The difference is that your Files & Uploads will be pointing at the URL you insert above for storage. Probably not worth messing with unless you have a real need for large numbers of large files, and even then you could probably just host those files elsewhere and use something better than Files & Uploads as an interface to them.
