Take pride in your HTML

Imagine: I am writing something directly into the text editor.

I’ve copied something across from another source, and then something else from somewhere else. The fonts are all a little different, so I select all and change the fonts for consistency back to the default Font Family. I also notice the colours look a bit different, so select the 'automatic black' font colour choice. Everything looks great.

... and then you look at the HTML editor view...

Now that is certainly NOT what you were expecting to see! Where did all these attributes come from? I thought I restored all the default settings?!

The somewhat annoying thing about the Rich Text editor is that it is all about formatting. It will often try too hard to make sure all the formatting from whence content came is retained, even if you don't need it to. Plus, if you try to restore default settings, this will simply add the new formatting requirements on top, as a completely new set of format attributes, rather than stripping out unwanted formatting as desired.

There are a number of reasons why this is bad:

  • In-line styles cause longer rendering time for a page to load
  • The code looks really messy - others will judge you and your code
  • It's really bad practice for accessibility reasons
  • If edX were to create site-wide style changes, you would be left with a load of manual overwrite lines that you'd need to clean up
  • Clean up will take FOREVER

Taking pride in your HTML appearance is not just about in-line styles; it's about creating content in the most useful and sustainable way first time round. Whilst it might seem like an overhead initially, trust us when we say it will certainly speed things up a lot in the future if you get into good habits now.
