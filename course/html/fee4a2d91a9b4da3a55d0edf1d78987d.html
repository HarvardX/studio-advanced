Tool Settings

This group covers components or modules that you can add to your edX course that aren't assignments.

Default: false

Other settings: true

Just like it says on the tin, this adds the ability to insert problems or tools that edX no longer supports. Depending on which version of edX you're using, this might include the older Poll module, the old LTI (rather than LTI Consumer) module, zooming image tools, the original drag-and-drop problem type, and more.

The advantage is that you have a few more options at your fingertips. The disadvantage is that anything you add might - no, probably will - be disabled by an update at some point in the future. In addition, as the page says, accessibility and internationalization are not always great for the older components. We can't recommend changing this setting.

Default: []

Other settings: A list of modules, in the format ["mod1", "mod2", "mod3"]

Enter the names of the advanced modules to use in your course.

This is one of our very favorite settings. Here you can activate things that are supported but, for reasons known only by the mystical edX developers themselves, hidden away from course creators in most courses.

Here's the list of what we recommend that you include in your course. Once you put a list of these into the Advanced Module List, you will see them appear under the "Advanced" button in Studio, just to the left of "Discussion" and "HTML".

  • "word_cloud": Creates a word cloud from a limited number of learner responses. Instruct your learners to enter just a word or two for maximum effectiveness. Great for seeing how learner attitudes change over time.
  • "lti_consumer": Allows you to insert tools based on the LTI version 1.1.1 specifications. For full detail, see the edX LTI component documentation. You cannot enable edX as an LTI provider from here, but your Open edX administrator might be willing to do that under certain circumstances.
  • "split_test": For randomized A/B testing. Check out the part of our course about designing content experiments.
  • "library_content": For randomized problem delivery.Check out the part of our course about building a content library.
  • "poll" and "survey": These create simple one-response or grid-response polls where learners can see the course totals. They're indispensible tools.
  • "drag-and-drop-v2": The updated, more accessible drag-and-drop exercise. You can get the full details on this at the edX Drag-and-Drop docs, but the short version is that your learners will drag an image (or bit of text) onto another image. It's a very flexible tool.
  • "ubcpi": An asynchronous peer instruction tool, built by the University of British Columbia. Learners answer a multiple-choice question, state their rationale, and then respond to the rationales of others. This is fully documented at the edX Peer Instruction tool docs.

Here it is in copy-paste-friendly format:

[
    "word_cloud",
    "lti_consumer",
    "split_test",
    "library_content",
    "poll",
    "survey",
    "drag-and-drop-v2",
    "ubcpi"
]

If you're using Open edX, you may have a different set of these, especially if your team has done a lot of customization or if you're running with an older distribution. You should check with your site administrator.

Default: false

Other settings: true

If true, students can use the Student Notes feature.

Why in the world does this default to false? We recommend setting this to true. If you have a strong pedagogical need to prevent learners from taking typed notes in your course - for instance, if you want them to take notes in one specific other manner - then I suppose you could leave this as false. Or if all your content is dynamically generated and the notes won't show up over them properly. But honestly, turn it to true.

Note (sorry) that notes only work in HTML components right now, not in problems or on video transcripts or anywhere else that text appears on the screen. Just in HTML.

Default: []

Other settings: other things

For HTML textbooks that appear as separate tabs in the course, enter the name of the tab (usually the title of the book) as well as the URLs and titles of each chapter in the book.

This is separate from the Textbooks feature in edX, which adds PDF-format books. This adds HTML pages as chapters in your textbook, which shows up as a tab at the top of the page. Here's the format:

[
  {"tab_title": "Textbook 1",
   "url": "https://www.example.com/thiscourse/book1/book1.html" },
  {"tab_title": "Textbook 2",
   "chapters": [
       { "title": "Chapter 1", "url": "https://www.example.com/thiscourse/book2/Chapter1.html" },
       { "title": "Chapter 2", "url": "https://www.example.com/thiscourse/book2/Chapter2.html" },
       { "title": "Chapter 3", "url": "https://www.example.com/thiscourse/book2/Chapter3.html" },
       { "title": "Chapter 4", "url": "https://www.example.com/thiscourse/book2/Chapter4.html" },
       { "title": "Chapter 5", "url": "https://www.example.com/thiscourse/book2/Chapter5.html" },
       { "title": "Chapter 6", "url": "https://www.example.com/thiscourse/book2/Chapter6.html" },
       { "title": "Chapter 7", "url": "https://www.example.com/thiscourse/book2/Chapter7.html" }
       ]
  }
]

Our sources inform us that "It is not a good idea to include a top-level URL and chapter-level URLs in the same textbook configuration." So, er, don't.

Default: [] (a blank list)

Other settings: A list in the form shown below.

Enter the passports for course LTI tools in the following format: "id:client_key:client_secret".

This is how you pass LTI security info to edX. Here's a sample, where the "ID" matches what you put into the LTI component itself:

[
  "id1:client_key1:client_secret1", 
  "id2:client_key2:client_secret2", 
  "id3:client_key3:client_secret3"
]

Default: false

Other settings: true

Enter true or false. When true, students can see the calculator in the course.

If you're running a maths course, turn this on. It's not as if your learners don't have access to all the calculation they could possibly desire anyway; your course is on the internet right next to Google and Wolfram Alpha. Let them use the edX calculator.
