🔀 Randomizing problems

Now let's get into some technological solutions. The fastest, easiest, and most reliable cheating methods all rely on having every problem be the same every time. Let's look at a few ways to avoid that.

Multiple choice

This is easy. Really, really easy. You should do it literally every time you write an MCQ. Here's how you do it:

>>I am the question prompt! Answer me!<<

(!) A wrong answer
(x) A right answer
( ) Another wrong answer
( ) More than four answers in an MCQ?? Surely it cannot be!?
(@) None of the above

[explanation]
Every problem should have an explanation.
[explanation]

Those three symbols, !, x, and @, have special functions:

  • ! randomizes the problem. Put ! in the parens for the first answer and students will see the results shuffled.
  • x says which answer is right, as per usual.
  • @ locks an answer in place. It stays "at" the location you select.
  • You can combine these just fine. (x!) works, as does (@x), and probably even (!@x) as long as it's the first choice.

Really, genuinely: unless you need your choices to be in order, you should randomize every time. It takes practically no time to throw an ! into your problems. Check the edX documentation for more detail on each of these, including the XML code and hwo to set up answer pools.

You can only shuffle Multiple-Choice, not Checkbox or Dropdown.
