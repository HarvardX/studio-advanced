How it Works

HX-JS seeks out the classes hx-sliderbox, hx-slider, hx-navslider, and hx-bigslider. The first is used as a container on all sliders. The second one is used for single sliders. The last two are for paired sliders.

Every first-level child within those divs gets turned into a slide. It doesn't matter whether you're making a slider of images, a slider of paragraphs, or a slider of divs, it all works the same way. Slick will wrap whatever the child is in a div tag, so that it displays as a block element.

You might have noticed the odd style element style="-webkit-user-select: auto; -moz-user-select: auto; -ms-user-select: auto; user-select: auto;". This allows the user to select within the slider, which is particularly useful when you're showing code, like in this one.

Options

You can set the options for your slider by declaring hxLocalOptions at the top of your raw HTML component:

var hxLocalOptions =  {
  // Default options for the Slick image slider
  slickOptions: {
    arrows: true,
    dots: true,
    infinite: true,
    slidesToShow: 3,
    slidesToScroll: 3
  },
  // Default options for image slider navigation
  slickNavOptions: {
    asNavFor: '.hx-bigslider',
    variableWidth: true,
    focusOnSelect: true,
    slidesToShow: 3,
    slidesToScroll: 1
  },
  // Default options for single big image slider paired to nav.
  slickBigOptions: {
    asNavFor: '.hx-navslider',
    arrows: false,
    dots: true,
    draggable: false,
    fade: true,
    adaptiveHeight: true,
    slidesToShow: 1,
    slidesToScroll: 1
  }

Need more options? Slick has a lot of them. Check out the settings for the Slick slider, all of which you can use here.

Caveats

One of each. You only get to have one slider and one paired slider per page. This is a limitation of HX-JS rather than a limitation of Slick, so you could get around this with your own implementation if you wanted to.

Download burden. Be kind to people in the world who are paying for bandwidth by the megabyte! Make small images for small sliders. In our examples we've sometimes used the same image for the top and bottom, which also helps to cut down on bandwidth. You can also use the "lazyLoad" option to keep large images from eating up bandwidth until a slide is actually requested by the user.

Troubleshooting

Height variations? Images of different sizes sometimes look awkward in the carousel (as you might have seen on the previous page). You might want to rescale, crop, or change the "adaptiveHeight" setting.

Unpaired sliders? Odds are you forgot to use the hx-bigslider and hx-navslider classes.

Weird formatting or effects, but not on slide #1. You're probably using the lazyLoad functionality. If your images aren't loaded when the page loads, then the javascript on your page won't be able to find them. If you're doing some sort of processing with those slides, Slick fires events when a slide loads, changes, is destroyed, etc. You can use those events to call your javascript on them.

Just not working? If you have custom settings, something broken there can keep other parts of the code from working. You might see only one part of a paired slider, or your entire slider might just spill out onto the page vertically. Double-check your formatting and look in the javascript console to see if there's an error you can find there.
