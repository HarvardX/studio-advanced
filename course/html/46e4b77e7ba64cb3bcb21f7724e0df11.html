If you've looked through Advanced Problem Types demo in this course, you've already seen a custom grading library at work!

The matching problem at the top of that page? Here's the grading code.

This line imports the grader itself. We've got a module called python_lib.zip (it has to be called that), and inside it there's an HXGraders function.

from python_lib import HXGraders

Here's the part of the code that's specific to this problem. It sets the answers and some options.

right_answer = []

right_answer.append( [['A','1'],['B','2'],['C','1']] )
right_answer.append( [['A','1'],['B','2']] )
right_answer.append( [['B','2'],['C','1']] )
right_answer.append( [['A','1']] )
right_answer.append( [['C','1']] )

partial_credit = True
feedback = True

Here's the part that does the grading.

def answercheck(e, ans):

  # This pulls from the python_lib.zip file in Files & Uploads.
  return HXGraders.matchingAGrader(ans, right_answer, partial_credit, feedback)

Yes, that is the entire part that does the grading. No lines and lines of if/then clauses; it just sends the student-submitted answer away to the library to handle the grading, and returns the result.

If you want to see the grading code itself, here's the whole package: python_lib.zip. Take that file, keep it zipped up, and drop it in your Files & Uploads.

You can find the source code at the HX-PY GitHub repository. The grading code is MIT-licensed, so you can use it in your own courses all you like. It handles grading for matching problems, ordering problems, range-guessing problems, and basically every unusual type you might have seen in different places in this course.

Warning: You Only Get One

We've mentioned it obliquely a few times, but just to state it outright: you can only have one grading library. This is because it must be named python_lib.zip. We don't know of any way around it. If you want to use multiple grading libraries, just unzip them all in the same folder, then zip them all up together again. Easy peasy! Just remember to go through this process every time you update the code, or you'll accidentally overwrite your big, full-featured library with a smaller one.
