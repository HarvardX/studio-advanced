Problem Setup

JSInput problems have several pieces. The files you need to build are in boldface.

  • The view that the learner sees is created by...
    • An HTML file, stored in Files & Uploads.
    • A css file, stored in Files & Uploads.
    • A Javascript file, stored in Files & Uploads.
  • Information that appears around that view is created by...
    • An XML file, entered into a Blank Advanced Problem.
    • The edX platform itself, which displays the Submit button, the number of available attempts, the learner's grade, etc.
  • Pieces that don't affect the learner's view, but can add functionality, might include...
    • jschannel.js, which allows files to be stored outside of Files & Uploads.
    • Data files that define the specific problem you're giving the learner. This might be written into the XML in Python, or stored in JS format in Files & Uploads.
    • Other javascript libraries, referenced in the HTML file and stored anywhere.
    • Grading code, written in Python. This could be stored in a grading library in Files & Uploads, or written directly into the XML file.

It's not quite as bad as it looks, but you can probably see why it takes a little work to set up.
