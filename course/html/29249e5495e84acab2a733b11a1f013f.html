Don't Repeat Yourself

If you're a programmer, you've probably heard the mantra "Don't Repeat Yourself", or DRY. (In fact, you've probably heard it so many times that it's become somewhat ironic...) The idea is that writing code in one place and referencing it or calling it in other places is not only faster in terms of raw keystrokes, but better for the structure of your programs.

The edX platform doesn't completely believe in the DRY principle, but this is one place where it's been implemented to a T.

Keeping your code in a grading library means that...

  • As mentioned before, you don't need to fix it in a dozen places if you have a dozen of the same problem type. You can just fix the library.
  • It's just plain easier to keep track of. You don't need to download the course tarbal and search through it painstakingly for every custom problem, you can just reupload a single file.
  • You can implement more complex code without creating a bloated 200-line-long XML file.
  • In fact, your code can get big enough to handle exceptions, corner cases, and even multiple options without breaking.
  • As you get better at programming, you can have fewer embarassments out in the wild. Not that your author would know anything about that, no.
