Interactive Walkthroughs


Easy

Hour

HTML

An interactive walkthrough provides a way to textually describe what learners are seeing on the screen. The text for the walkthrough appears next to these components, and can be stepped through using mouse or keyboard.

Interactive walkthroughs are great for pointing out parts of the edX interface to newcomers, indicating important parts of a complex table or code structure, or instructing learners on how to use more complex interactives. They're not great for use with videos, as no time-based information is included. If you want to add information to a video, consider using Video Pop-up Links instead.

The goal: Put an interactive walkthrough into your course to indicate specific landmarks on a page.
Prerequisites: Familiarity with HTML to use or for minor customization. Familiarity with Javascript to edit.
Project length: 30 minutes to learn, 20 minutes to implement, 10 minutes to test.
Ingredients:

  • intro_accessible.js, a fork from Intro.js 2.0 that adds minor accessibility improvements. Download this and place it in Files & Uploads.
  • introjs.css, the style sheet. Download this and place it in Files & Uploads.
  • walkthrough_snippet.html, a piece of HTML that will enable the walkthrough on one particular page. Copy from this file (or from the code you see on the next page) and paste it into a Raw HTML component or an Update.
